It is possible to specify a list of VS Code extensions in the devcontainer.json file that will be installed when the Codespace is created. This can be useful for presenting the user of the Codespace with a pre-configured environment to aid their productivity.

Syntax

To specify VS Code extensions in the .devcontainer/devcontainer.json file, create a key-value pair within the highest-level object with the key customizations and a value of an object. Within this object, create another key-value pair with the key vscode whose value is an object. Within this object, create an extensions key with an array of extension identifiers as the value. Each identifier should be expressed as a string. For example, your devcontainer.json may look as follows:

{
  "image": "mcr.microsoft.com/devcontainers/base:alpine",
  "name": "Example",
  "customizations": {
    "vscode": {
      "extensions": [
        "ms-python.python",
        "ms-python.flake8"
      ]
    }
  }
}

In the example above, the extensions Python and Flake8 will be installed when the Codespace is created.

Mini-Example

Click the link at the top of the page to open a Codespace. Wait for the Codespace to load. Open the Extensions view. You should see some default extensions installed, as well as:

  • Preview PDF
  • Pink Candy Theme

Note that the Pink Candy theme will be installed, but not applied automatically. You can activate it from its page that you can access in the Extensions view by clicking "Set Color Theme".

You can view the .devcontainer/devcontainer.json file to see how the extensions are specified.
