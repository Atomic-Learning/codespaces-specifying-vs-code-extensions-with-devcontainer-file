It is possible to specify a list of VS Code extensions in the devcontainer.json file that will be installed when the Codespace is created. This can be useful for presenting the user of the Codespace with a pre-configured environment to aid their productivity.

Syntax

To specify VS Code extensions in the .devcontainer/devcontainer.json file, create a key-value pair within the highest-level object the key extensions and a value of another object. Within this object, create another key-value-pair with the key vscode whose value is an array. Within this array, you can list the identifiers of the extensions you want to install, each expressed as a string. For example, your devcontainer may look as follows:

{
  "image": "mcr.microsoft.com/vscode/devcontainers/ubuntu:latest",
  "name": "Example",
  "extensions": {
    "vscode": [
      "ms-python.python",
      "ms-python.flake8"
    ]
  }
}

In the example above, the extensions Python and Flake8 will be installed when the Codespace is created.

Mini-Example

Click the link at the top of the page to open a Codespace. Wait for the Codespace to load. Open the Extensions view. You should see some default extensions installed, as well as:

  • Preview PDF
  • Pink Candy Theme

Note that the Pink Candy theme will be installed, but not applied automatically. You can activate it from its page that you can access in the Extensions view.

You can view the .devcontainer/devcontainer.json file to see how the extensions are specified.
